Documentation for src/dataset.py
================================

Purpose of this file
--------------------

The file src/dataset.py contains all dataset-related code for the RadioML
2018.01A Automatic Modulation Classification project.

Its main responsibilities are:

1. Define the 24 modulation class names.
2. Read labels and SNR values from the RadioML HDF5 file.
3. Create train/validation/test split index files.
4. Load split indices for later experiments.
5. Optionally limit split sizes for quick experiments.
6. Provide a PyTorch Dataset class that reads I/Q samples lazily from HDF5.
7. Build PyTorch DataLoader objects for training, validation, and testing.
8. Provide a command-line interface for creating split files.

The most important idea is that the full RadioML signal array is very large.
The project does not load the entire X array into memory. Instead, it stores
only row indices for train/validation/test splits, then loads each sample from
the HDF5 file only when PyTorch asks for it.


RadioML 2018.01A dataset structure
----------------------------------

The raw dataset file is:

    data/raw/radioml2018/GOLD_XYZ_OSC.0001_1024.hdf5

It contains three main arrays:

    X: signal samples
    Y: modulation labels
    Z: SNR values

Expected shapes:

    X: (2555904, 1024, 2)
    Y: (2555904, 24)
    Z: (2555904,)

Meaning:

    2555904 = number of examples
    1024    = number of time samples per signal
    2       = I and Q channels
    24      = modulation classes

For a single index i:

    X[i] = one I/Q signal sample
    Y[i] = one-hot modulation label
    Z[i] = SNR value in dB

Example:

    X[100] might contain one noisy QPSK signal.
    Y[100] tells the project that the signal belongs to QPSK.
    Z[100] tells the project the SNR, for example 10 dB.

Function: _validate_split_ratios
--------------------------------

Signature:

    def _validate_split_ratios(
        train_split: float,
        val_split: float,
        test_split: float,
    ) -> None:

Purpose:

This helper checks that the train/validation/test split ratios are valid.

Example valid split:

    train_split = 0.6
    val_split   = 0.2
    test_split  = 0.2

The total is:

    0.6 + 0.2 + 0.2 = 1.0

The function checks two conditions:

1. The three split ratios must sum to 1.0.
2. All split ratios must be greater than 0.

Code:

    total = train_split + val_split + test_split

This adds the three floating-point values.

    if not np.isclose(total, 1.0):

NumPy method:

    np.isclose(a, b)

This checks whether two floating-point numbers are approximately equal.
Floating-point arithmetic can have tiny precision errors, so this is better
than using:

    total == 1.0

Example:

    np.isclose(0.999999999, 1.0) -> True

If the total is not close to 1.0, the function raises:

    raise ValueError(...)

Python exception:

    ValueError

This is used when a function receives a value that has the correct type but an
invalid value.

The formatted string:

    f"(got {total:.6f})"

prints the total with 6 digits after the decimal point.

Then the function checks:

    if min(train_split, val_split, test_split) <= 0:

Python built-in:

    min(...)

This returns the smallest value. If the smallest split is less than or equal to
zero, at least one split is invalid.

The function returns None because it is only a validation function. If
something is wrong, it raises an error. If everything is correct, it silently
finishes.

Function: stratified_train_val_test_split
-----------------------------------------

Signature:

    def stratified_train_val_test_split(
        indices: np.ndarray,
        stratify_key: np.ndarray,
        train_split: float,
        val_split: float,
        test_split: float,
        random_seed: int,
    ) -> tuple[np.ndarray, np.ndarray, np.ndarray]:

Purpose:

This function creates train/validation/test splits while preserving the
distribution of groups.

In this project, the groups are usually class+SNR combinations.

For example:

    OOK at -20 dB
    OOK at -18 dB
    QPSK at 0 dB
    256QAM at 30 dB

Each group is split separately, then all group pieces are combined. This keeps
the train/validation/test splits balanced.

Inputs:

    indices

Array of all row numbers in the dataset.

Example:

    [0, 1, 2, 3, ..., N-1]

    stratify_key

Array with one group identifier for each sample.

If two samples have the same stratify_key, they belong to the same group.

    train_split, val_split, test_split

Fractions for splitting each group.

    random_seed

Seed for reproducible random shuffling.

Code:

    rng = np.random.default_rng(random_seed)

NumPy method:

    np.random.default_rng(seed)

This creates a modern NumPy random number generator.

Using a seed makes the result reproducible. If the same seed and same data are
used, the same split is created again.

    train_parts: list[np.ndarray] = []
    val_parts: list[np.ndarray] = []
    test_parts: list[np.ndarray] = []

These lists temporarily store split pieces for each group.

Then:

    for key in np.unique(stratify_key):

NumPy method:

    np.unique(array)

This returns all unique values in the array.

If stratify_key represents class+SNR groups, this loop processes one class+SNR
group at a time.

Inside the loop:

    group_indices = indices[stratify_key == key].copy()

Important NumPy operations:

    stratify_key == key

This creates a boolean mask. It is True where the sample belongs to the current
group.

Example:

    [False, True, False, True]

    indices[mask]

This selects only indices where the mask is True.

    .copy()

This creates a separate copy of the selected indices. That is useful because
the code then shuffles the group indices.

    rng.shuffle(group_indices)

NumPy random method:

    rng.shuffle(array)

This shuffles the array in place. "In place" means it modifies the existing
array instead of returning a new one.

Then:

    n = len(group_indices)

Python built-in:

    len(...)

Returns the number of elements in the current group.

    n_test = int(round(n * test_split))
    n_val = int(round(n * val_split))

Python built-ins:

    round(...)
    int(...)

round computes the nearest integer value. int converts it to an integer type.

Then the shuffled group is split:

    test_parts.append(group_indices[:n_test])

This takes the first n_test indices for the test split.

List method:

    list.append(value)

Adds one item to the end of a list.

Slicing:

    group_indices[:n_test]

Means "from the start up to n_test, not including n_test".

Then:

    val_parts.append(group_indices[n_test : n_test + n_val])

This takes the next n_val samples for validation.

Then:

    train_parts.append(group_indices[n_test + n_val :])

This takes all remaining samples for training.

After all groups are processed:

    train_idx = np.concatenate(train_parts)
    val_idx = np.concatenate(val_parts)
    test_idx = np.concatenate(test_parts)

NumPy method:

    np.concatenate(list_of_arrays)

This joins many arrays into one array.

Then:

    rng.shuffle(train_idx)
    rng.shuffle(val_idx)
    rng.shuffle(test_idx)

This shuffles the final train, validation, and test arrays so that samples from
the same group are not kept next to each other.

Finally:

    return train_idx, val_idx, test_idx


Function: create_split_indices

Signature:

    def create_split_indices(

Purpose:

Split ratios.

    random_seed

Seed for reproducible splitting.

Step 1:

Step 2:

    output_path = Path(output_path)

Path method:

    Path(...)

Converts strings or existing Path objects into Path objects.

Step 3:

    output_path.parent.mkdir(parents=True, exist_ok=True)

Path property:

    output_path.parent

Returns the directory containing the file.

Example:

    Path("data/splits/file.npz").parent -> Path("data/splits")

Path method:

    mkdir(...)

Creates a directory.

Arguments:

    parents=True

Creates parent directories too if they do not exist.

    exist_ok=True

Does not raise an error if the directory already exists.

Step 4:

This reads integer labels and SNR values.

Step 5:

        raise ValueError(...)

This verifies that each label has a corresponding SNR value.

Step 6:

    indices = np.arange(len(y), dtype=np.int64)

NumPy method:

    np.arange(stop, dtype=...)

Creates an array of numbers:

    [0, 1, 2, ..., stop - 1]

If there are N samples, this creates all row indices from 0 to N-1.

Step 7:

    if stratify_by_snr:
        snr_levels, snr_codes = np.unique(z, return_inverse=True)
        stratify_key = y * len(snr_levels) + snr_codes.reshape(-1)
    else:
        snr_levels = np.unique(z)
        stratify_key = y

NumPy method:

    np.unique(z, return_inverse=True)

With return_inverse=True, NumPy returns two things:

    snr_levels = sorted unique SNR values
    snr_codes  = integer code for each sample's SNR

Example:

    z = [-20, -20, -18, -16]

Then:

    snr_levels = [-20, -18, -16]
    snr_codes  = [0, 0, 1, 2]

The line:

    stratify_key = y * len(snr_levels) + snr_codes.reshape(-1)

creates a unique group ID for each class+SNR combination.

Example:

    number of SNR levels = 26
    class y = 4
    snr code = 10

Then:

    stratify_key = 4 * 26 + 10 = 114

All samples with class 4 and SNR code 10 get the same key.

If stratify_by_snr is False:

    stratify_key = y

Then the split is balanced only by modulation class, not by SNR.

Step 8:

    train_idx, val_idx, test_idx = stratified_train_val_test_split(...)

This creates the actual split index arrays.

Step 9:

    metadata = {...}

This dictionary describes the split.

Python dictionary:

    key: value

The metadata includes:

    source_hdf5
    num_samples
    train_split
    val_split
    test_split
    random_seed
    stratify_by
    train_samples
    val_samples
    test_samples
    snr_levels

Conversions used:

    int(...)
    float(...)
    str(...)

These convert NumPy values into normal Python values so they can be stored as
JSON.

    snr_levels.tolist()

NumPy method:

    array.tolist()

Converts a NumPy array into a Python list.

Step 10:

    np.savez_compressed(
        output_path,
        train_idx=np.sort(train_idx),
        val_idx=np.sort(val_idx),
        test_idx=np.sort(test_idx),
        metadata=json.dumps(metadata, indent=2),
    )

NumPy method:

    np.savez_compressed(file, name1=array1, name2=array2, ...)

Saves multiple arrays into one compressed .npz file.

This project saves:

    train_idx
    val_idx
    test_idx
    metadata

NumPy method:

    np.sort(array)

Sorts the indices before saving.

Sorting can improve HDF5 read locality because nearby row indices are closer
together in the file. The training DataLoader still shuffles training samples
later, so sorting here does not remove training randomness.

json method:

    json.dumps(metadata, indent=2)

Converts a Python dictionary to a JSON-formatted string. indent=2 makes it
human-readable.

Step 11:

    return output_path

The function returns the location of the saved split file.

Method: RadioMLDataset.__getitem__
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Signature:

    def __getitem__(
        self,
        idx: int,
    ) -> tuple[torch.Tensor, torch.Tensor, torch.Tensor]:

Purpose:

Loads one sample from the HDF5 file and returns it as PyTorch tensors.

PyTorch calls this method automatically when building batches.

Input:

    idx

The local index inside this dataset.

Important distinction:

    idx        = position inside train/val/test subset
    sample_idx = actual row number inside the full HDF5 file

Code:

    sample_idx = int(self.indices[idx])

This converts the selected split index into a normal Python integer.

Then:

    x = self.file["X"][sample_idx].astype(np.float32).T

This reads one signal sample.

Breakdown:

    self.file

Gets the HDF5 file handle.

    self.file["X"]

Gets the HDF5 dataset named X.

    self.file["X"][sample_idx]

Reads one row from X.

Original shape:

    (1024, 2)

This means:

    1024 time samples
    2 I/Q components

    .astype(np.float32)

Converts the sample to 32-bit floating point. PyTorch neural networks normally
use float32.

    .T

NumPy property:

    array.T

Returns the transpose of the array.

Shape changes from:

    (1024, 2)

to:

    (2, 1024)

This is required because PyTorch Conv1d expects:

    (channels, sequence_length)

For a batch, the DataLoader creates:

    (batch_size, channels, sequence_length)

In this project:

    (batch_size, 2, 1024)

        std = x.std()
        if std > 0:
            x = (x - x.mean()) / std

NumPy methods:

    x.std()

Computes the standard deviation of all values in the sample.

    x.mean()

Computes the mean of all values in the sample.

The formula:

    x = (x - x.mean()) / std

standardizes the sample to approximately zero mean and unit standard deviation.

The check:

    if std > 0:

prevents division by zero.

Then the label is loaded:

Breakdown:

    self.file["Y"][sample_idx]

Reads the one-hot label vector for this sample.

    np.argmax(...)

Finds the position of the maximum value. Since the label is one-hot, this
returns the class index.

    int(...)

Converts the NumPy integer to a Python integer.

Then the SNR is loaded:

Breakdown:

    self.file["Z"][sample_idx]

Reads the SNR value for this sample.

    np.asarray(...)

Converts the value into a NumPy array. This makes the next reshape operation
safe even if h5py returns a scalar-like object.

    .reshape(-1)

Flattens it into a one-dimensional array.

    [0]

Gets the first value.

    float(...)

Converts it to a normal Python float.

Finally, the method returns:

    return (
        torch.from_numpy(x),
        torch.tensor(y, dtype=torch.long),
        torch.tensor(snr, dtype=torch.float32),

PyTorch methods:

    torch.from_numpy(x)

Creates a PyTorch tensor that shares memory with the NumPy array when possible.
This is efficient.

    torch.tensor(y, dtype=torch.long)

Creates a tensor from the integer label.

The dtype torch.long is required by PyTorch CrossEntropyLoss, which expects
class labels as integer class indices, not one-hot vectors.

    torch.tensor(snr, dtype=torch.float32)

Creates a float32 tensor for the SNR value.

Returned sample:

    x   -> shape (2, 1024), dtype float32
    y   -> scalar integer tensor, dtype long

__del__ is a cleanup backup. It is useful, but in complex multiprocessing
settings, explicit cleanup is still better when possible.

How this file is used by training
---------------------------------

In src/train.py, the training script calls:

    train_loader, val_loader, _test_loader = build_dataloaders(config)

This uses the config file to:

1. Load train/validation/test indices.
2. Optionally reduce them for quick experiments.
3. Create RadioMLDataset objects.
4. Create PyTorch DataLoader objects.

During training, a batch has:

    x:   (batch_size, 2, 1024)
    y:   (batch_size,)
    snr: (batch_size,)

The training loop uses:

    x
    y

It ignores:

    snr

because SNR is not an input to the model.


How this file is used by evaluation
-----------------------------------

In src/evaluate.py, the evaluation script also calls:

    _train_loader, _val_loader, test_loader = build_dataloaders(config)

During evaluation, the script keeps:

    x
    y
    snr

The model predicts from x.

The prediction is compared against y.

The SNR value is used to calculate per-SNR accuracy.

This is why __getitem__ returns SNR even though training does not use it.

Important design decisions in this file
---------------------------------------

1. HDF5 is read lazily

The full X array is not loaded into memory. Only one sample is loaded when
needed.

2. Split files store indices, not data

The .npz split file is small because it stores only row numbers.

3. Splits are stratified

By default, the split preserves both class and SNR distributions.

This is important for scientific evaluation because each split should contain
similar amounts of every modulation class at every SNR.

4. PyTorch receives channels-first data

RadioML stores each sample as:

    (1024, 2)

The dataset returns:

    (2, 1024)

because Conv1d expects:

    channels first

5. SNR is returned even though it is not used for training

SNR is required for analysis:

    accuracy per SNR
    low/mid/high SNR comparison
    SNR-vs-accuracy plots


Things to improve later
-----------------------

The current sample_normalize option uses standardization:

    x = (x - x.mean()) / x.std()

For signal processing, RMS normalization may be more meaningful:

    rms = sqrt(mean(x ** 2) + epsilon)
    x = x / rms

Possible future input feature variants:

    I, Q
    I, Q, amplitude
    I, Q, amplitude, sin(phase), cos(phase)

Where:

    amplitude = sqrt(I ** 2 + Q ** 2)
    phase = atan2(Q, I)
    sin_phase = sin(phase)
    cos_phase = cos(phase)

Those changes would also belong in RadioMLDataset.__getitem__, because that is
where one raw sample is transformed before being returned to the model.
